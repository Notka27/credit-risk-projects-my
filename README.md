## Проект 1: Credit Risk Analysis — прогнозирование дефолта заемщиков (Kaggle)

Проект по кредитному скорингу: предсказание вероятности невозврата кредита на основе данных о заемщике.

### Цель проекта

Построить и сравнить модели машинного обучения для прогнозирования дефолта (loan_status = 1). Выявить ключевые факторы, влияющие на риск невозврата кредита.

### Данные

- **Источник:** Kaggle — Credit Risk Dataset
- **Объем:** 32 581 запись, 12 признаков
- **Целевая переменная:** `loan_status` (0 — платежеспособен, 1 — дефолт)

#### Признаки

| Признак | Описание |
|---------|----------|
| person_age | Возраст заемщика |
| person_income | Годовой доход |
| person_home_ownership | Тип жилья (аренда, собственность, ипотека) |
| person_emp_length | Трудовой стаж (лет) |
| loan_intent | Цель кредита |
| loan_grade | Кредитный класс (A-G) |
| loan_amnt | Сумма кредита |
| loan_int_rate | Процентная ставка |
| loan_percent_income | Отношение суммы кредита к доходу |
| cb_person_default_on_file | Наличие дефолта в прошлой кредитной истории |
| cb_person_cred_hist_length | Длина кредитной истории (лет) |

### Исследовательский анализ данных (EDA)

#### Ключевые выводы

- **Дефолт чаще встречается у заемщиков 20–30 лет**
- **Риск резко возрастает** при:
  - процентной ставке > 15%
  - отношении кредита к доходу > 0.3
  - кредитной истории < 5 лет
  - трудовом стаже < 10 лет
- **Безопасные категории:**
  - жилье в собственности (lowest risk)
  - цель кредита — бизнес (lowest default rate)
  - кредитный класс A-B
- **Высокий риск:**
  - аренда жилья
  - цели: объединение долгов, ремонт, медицина
- **Дисбаланс классов:** 22% дефолтов, 78% платежеспособных

#### Корреляционный анализ

| Признаки | Корреляция |
|----------|-----------|
| Возраст / длина кредитной истории | 0.86 (мультиколлинеарность, для деревьев не критично) |
| Сумма кредита / отношение кредита к доходу | 0.57 |
| Остальные признаки | -0.25 .. 0.38 |

### Предобработка данных

1. **Обработка выбросов** — ограничение 99-м процентилем (удалено ~16% данных)
   - Возраст > 80 лет
   - Доход > 99-го процентиля
   - Стаж > 99-го процентиля
   - Сумма кредита > 99-го процентиля
   - Процентная ставка > 99-го процентиля
   - Отношение кредита/доход > 99-го процентиля
2. **Заполнение пропусков** — медианой (`person_emp_length`, `loan_int_rate`)
3. **Кодирование категориальных признаков** — `LabelEncoder`
4. **Масштабирование числовых признаков** — `StandardScaler`
5. **Разделение данных** — train 80% / test 20% (стратификация по целевому признаку)

### Модели

| Модель | AUC-ROC | Точность (Accuracy) | Recall (дефолт) | Precision (дефолт) |
|--------|---------|--------------------|-----------------|-------------------|
| Логистическая регрессия | 0.846 | 0.85 | 0.43 | 0.71 |
| **Random Forest** | **0.933** | **0.93** | **0.69** | **0.97** |
| Gradient Boosting | 0.929 | 0.92 | 0.67 | 0.93 |

#### Лучшая модель — Random Forest

- **Точность предсказания дефолта (precision):** 97%
- **Обнаруживает реальных дефолтов (recall):** 69%
- **Ошибки:** 25 ложноположительных, 349 ложноотрицательных

### Важность признаков (Random Forest)

| Признак | Важность |
|---------|----------|
| loan_grade (кредитный класс) | 0.221 |
| person_income (доход) | 0.146 |
| loan_intent (цель кредита) | 0.117 |
| cb_person_default_on_file (дефолт в прошлом) | 0.115 |
| loan_int_rate (процентная ставка) | 0.092 |
| person_emp_length (стаж) | 0.076 |
| loan_percent_income (кредит/доход) | 0.071 |
| person_home_ownership (тип жилья) | 0.067 |
| person_age (возраст) | 0.045 |
| loan_amnt (сумма кредита) | 0.036 |
| cb_person_cred_hist_length (длина кредитной истории) | 0.012 |

### Результаты и интерпретация

#### Ключевые факторы дефолта

- **Низкий кредитный класс (loan_grade)**
- **Низкий доход**
- **Цель кредита** — объединение долгов, ремонт, медицинские расходы
- **Наличие дефолта в прошлом**
- **Высокая процентная ставка (>15%)**
- **Высокое отношение кредита к доходу (>0.3)**

## Проект 2: Bondora Risk Analysis — реальные P2P-кредитные данные

Проект по кредитному риск-анализу на реальных данных европейской P2P-платформы Bondora. Главный вызов — работа с «грязными» данными (пропуски до 90%, 112 признаков) и предотвращение data leakage.

### Цель проекта

Построить честную модель предсказания дефолта заемщиков на основе данных, доступных на момент выдачи кредита (без утечек данных из будущего).

### Данные

- **Источник:** Bondora (реальные P2P-кредитные данные)
- **Объем:** 134 529 записей, 112 признаков
- **Целевая переменная:** `loan_default` (0 — платежеспособен (Current/Repaid), 1 — дефолт (Late))
- **Пропуски:** до 90% в некоторых колонках (кредитные скоринги, стаж работы, иждивенцы)

### Проблема data leakage (утечка данных)

В исходных данных присутствовали признаки, отражающие **уже наступившую просрочку платежей**
- `ActiveLateLastPaymentCategory_days`
- `WorseLateCategory_days`

Эти признаки не известны в момент выдачи кредита. Модель с ними показывала нереалистично высокий AUC-ROC (0.995). После их удаления получена **честная оценка** качества модели.

### Предобработка данных

1. **Отбор признаков** — из 112 колонок отобраны 35 релевантных (демография, доходы, кредитные скоринги, параметры кредита)
2. **Заполнение пропусков:**
   - `NrOfDependants` — нулем
   - `WorkExperience` — преобразование категорий в числа + группировка по возрасту
   - Кредитные скоринги — группировка по стране + медиана
   - Категориальные признаки — модой
   - `DebtToIncome`, `FreeCash` — медианой
3. **Преобразование категориальных кредитных скорингов в числовые шкалы:**
   - `CreditScoreEsEquifaxRisk` (AAA→1, D→6)
   - `CreditScoreEsMicroL` (M1→1, M10→10)
   - `CreditScoreFiAsiakasTietoRiskGrade` (RL0→0, RL5→5)
4. **Кодирование категориальных признаков** — `LabelEncoder`
5. **Масштабирование числовых признаков** — `StandardScaler`

### Результаты

| Версия модели | AUC-ROC | Комментарий |
|---------------|---------|-------------|
| С утечками (включая просрочки) | 0.995 | модель с утечкой данных (признаки просрочки известны после выдачи кредита)|
| **Без утечек (только честные признаки)** | **0.809** | модель без утечки данных |

#### Метрики модели без учета признаков просрочки платежей 

| Показатель | Значение |
|------------|----------|
| Accuracy | 0.75 |
| Precision (дефолт) | 0.68 |
| Recall (дефолт) | 0.50 |
| AUC-ROC | **0.809** |
